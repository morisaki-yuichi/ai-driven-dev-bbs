# AI駆動開発体験・教材プロジェクト

本プロジェクトは、人間が直接コードを書かずにAIエージェントを指揮してソフトウェアを開発する「AI駆動開発 (AI-Driven Development)」を体験するための教材用テンプレートです。

---

## 1. 【人間向け】教材利用ガイド

学習者（人間）は、エンジニアではなく「AIエージェントの指揮者」として振る舞います。

### 学習の目的

- AIエージェントに正確な指示（プロンプト）を出す能力を養う。
- AIが自律的に動けるよう、フロー整備やSKILL定義を行う能力を養う。
- 直接手を下さずに、AIをデバッグや実行に導く「AIエージェントエンジニアリング」を体験する。

### 守るべきルール

本教材を利用する際は、人間がソースコードやGitを操作することは禁止されています。詳細は以下のルール定義書を必ず確認してください。

- **[開発ルール定義書](./docs/guide/development_rules.md)**

---

## 2. 【AI/人間向け】アプリケーション開発仕様

本プロジェクトで開発の対象となるのは、ブラウザ上で動作する「掲示板（BBS）アプリケーション」です。

### プロダクト概要

ユーザー登録・ログインを行い、スレッドの作成やコメント投稿ができるシンプルな掲示板です。詳細は以下のドキュメントを参照してください。

- **[プロダクト技術仕様書](./docs/product/technical_spec.md)**: 技術スタックの選定指針や前提条件。
- **[要件定義 (Issues)](./docs/product/issues/)**: 各機能の詳細な仕様と受け入れ基準（AC）。
- **[画面設計 (Designs)](./docs/product/designs/)**: UIの構成、共通レイアウト、画面遷移の定義。
- **[テストシナリオ (Scenarios)](./docs/evaluation/scenarios/)**: 最終評価時にパスすべきE2Eテストの合格基準。

---

## 3. プロジェクトの評価と動作条件

本プロジェクトの成果物は、提出後に評価者がAIエージェントを用いて以下の指示（プロンプト）を与えることで初期化・動作確認が行われます。

### 初期化と起動の自動化

評価者は以下のプロンプトを用いて、自律的に動くAIエージェントにセットアップを指示します。学習者は、**以下の指示を受け取ったAIエージェントが、人間の手助けなくセットアップと起動を完了できること**を目標に実装・文書化を行ってください。

#### 評価時の初期セットアップ指示

> このプロジェクトのREADME.mdを読み込み、アプリケーションの初期セットアップ方法を把握してください。README.mdから参照されているドキュメントやスクリプトがあれば、それらも参照してください。
> その後、手順に従って初回のセットアップを実行してください。
> ユーザーが書き込むべき秘匿キーなどがある場合は、その旨をユーザーに伝えてください。それ以外はすべてAIが自動で実行してください。
> 初回セットアップが完了したら、その旨をユーザーに伝えてください。

#### 評価時のアプリケーション起動指示

> このプロジェクトのREADME.mdを読み込み、アプリケーションの起動方法を把握してください。README.mdから参照されているドキュメントやスクリプトがあれば、それらも参照してください。
> その後、手順に従ってアプリケーションを起動してください。
> 起動が完了したら、その旨をユーザーに伝えてください。

### テストシナリオによる機能検証

初期化・起動の確認後、評価者は [プロジェクト評価用 E2Eテストシナリオ](./docs/evaluation/scenarios/) に格納されている各シナリオをAIエージェントに実行させることで、各機能が要件通りに動作するかを検証します。

学習者は、**評価用のAIエージェントが自然言語で記述されたテスト手順を自律的に理解し、すべてのテストをパスできる状態**にプロジェクトを仕上げてください。
