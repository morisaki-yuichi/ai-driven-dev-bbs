# 開発ルール定義書 (Development Rules)

本プロジェクトは「AI駆動開発」を体験するための教材です。開発効率や品質の向上ではなく、AIエージェントにどこまで自律的に開発を任せられるかを試すことを目的としています。そのため、人間（ユーザー）の関与には制約が設けられています。

## 1. 開発時の禁止事項

以下の操作は、いかなる場合も人間が行うことを禁止します。

- **ソースコードの直接編集**: バグ修正、リファクタリング、新機能の実装など、一切のコード編集はAIに依頼してください。
- **Gitの直接操作**: コミット、プッシュ、プル、ブランチの作成や切り替えなどのGit操作はすべてAIに行わせてくだい。
- **開発ドキュメントの編集**: `docs/` 配下の要件定義や設計書の修正も（AIへのフィードバックを除き）人間が行うことは避けてください。

## 2. 人間が担当する役割

人間は「エンジニア」としてではなく、「AIエージェントのエンジニアリング（指示・管理）」に専念してください。

- **AIへの指示**: 開発の優先順位、機能の追加依頼、バグ発生時の報告。
- **エージェント設定の最適化**: `.AGENTS.md`、`.CLAUDE.md`、`.github/` や `.claude/` 以下のSKILL定義ファイルなど、AIの動作を制御するファイルの編集。
- **環境設定の編集**: エディタの設定、隠しファイルの調整など。
- **技術選定の評価**: [技術仕様書 (TECHNICAL_SPEC.md)](../product/technical_spec.md) に基づき、AIエージェントが提案する環境構成やライブラリ選定が適切かを判断・フィードバックしてください。
- **コマンドの実行**: 最終的な実行権限は人間にありますが、可能な限りAIにコマンドを提案させ、それを承認する形式をとってください。

## 3. AIが行き詰まった（スタックした）際の対応

AIが同じエラーを繰り返したり、どうしても解決できない問題に直面したりした場合は、以下の手順を踏んでください。

1. **デバッグの許可**: 問題箇所に限り、人間がデバッグやコードの修正を行ってもよいものとします。
2. **AIへのフィードバック（重要）**: 人間が修正を行った場合、必ずその「原因」と「解決策」をAIに伝えてください。
3. **再発防止策の実施**: 同じ問題が再発しないよう、AIにルール・SKILLの追加やドキュメントの更新を指示することを強く推奨します。
