# プロジェクト評価用 E2Eテストシナリオ

本ディレクトリのシナリオは、プロジェクトの最終評価時に評価者がAIエージェントを用いて実行する検証手順です。

学習者は、開発時にこれらのシナリオをそのまま実行する必要はありませんが、**最終的にこれらすべてのシナリオがパスするようにアプリケーションを構築すること**が求められます。評価時には、以下の前提条件に基づいてAIエージェントが自律的にテストを遂行します。

## 評価時の前提条件

1. **DBの初期状態**:
   - すべてのテストを開始する直前は、データベースの内容が完全に空（クリーン）な状態である必要があります。

2. **実行順序**:
   - テストシナリオは、ファイル名の先頭にある番号（`01_`, `02_`, ...）の順に実行してください。
   - シナリオ間ではデータベースのクリアを行わず、前のテストで作成されたユーザーやデータをそのまま利用します。

3. **データの作成**:
   - テストに必要なユーザーやスレッド、コメントなどのデータは、シナリオの中で手順として作成されます。事前のシードデータの投入は不要です。

## シナリオ一覧
- **01_authentication**: ユーザー登録、ログイン、ログアウトの検証
- **02_thread_management**: スレッド作成、権限による削除制限、一覧表示の検証
- **03_comment_management**: コメント投稿、削除時の文言置換、削除制約の検証
- **04_search_and_sort**: 検索、スクロール連携、ページネーション、ソート機能の検証
- **05_user_profile**: プロフィールの表示名変更と、全投稿への反映検証
