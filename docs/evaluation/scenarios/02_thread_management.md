# シナリオ02：スレッドの管理（作成・削除・一覧表示）

このシナリオでは、スレッドの作成から削除、および一覧表示でのメタデータの正確性をテストします。
※前のシナリオで作成したユーザー `testuser_01` でログイン済みであることを前提とします。

## 1. スレッド作成のテスト
1. `testuser_01` でログインする。
2. 「新規スレッド作成」ボタンをクリックする。
3. 以下の内容を入力して作成する。
    - タイトル: `AI駆動開発の未来について`
    - 本文: `AIエージェントによる自動プログラミングはどこまで進化するでしょうか？`
4. スレッド一覧画面に遷移し、今作成したスレッドが表示されていることを確認する。
5. 一覧画面で以下の項目が正しく表示されていることを確認する。
    - タイトル、本文（冒頭）、作成者（テストユーザー01）、コメント数（0）、作成日時。

## 2. 権限管理と削除のテスト
1. 別のブラウザ、または一度ログアウトして別のユーザー `testuser_02` で登録・ログインする。
2. 「AI駆動開発の未来について」のスレッド詳細画面を開く。
3. 自分の作成したスレッドではないため、削除ボタンが表示されないことを確認する。
4. `/スレッドID/delete` などのURLに直接アクセスしようとしても、エラーまたはリダイレクトされることを確認する。
5. `testuser_01` でログインし直す。
6. 「AI駆動開発の未来について」のスレッド詳細画面を開く。
7. 「スレッドを削除する」ボタンが表示されていることを確認し、クリックする。
8. 削除の確認ダイアログ（あれば）に同意し、一覧画面へリダイレクトされることを確認する。
9. 一覧画面から該当のスレッドが消えていることを確認する。
10. 削除されたスレッドのURLに直接アクセスし、404エラー（またはそれに準ずる表示）になることを確認する。
