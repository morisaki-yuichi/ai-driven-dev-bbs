# シナリオ01：ユーザー登録とログイン・ログアウト

このシナリオでは、新規アカウントの作成、バリデーションの確認、および基本的な認証フロー（ログイン・ログアウト）をテストします。

## 1. ユーザー登録のテスト
1. ブラウザでアプリケーションのトップページにアクセスする。
2. ログイン画面にリダイレクトされることを確認する。
3. 「新規登録」リンクをクリックして登録画面へ移動する。
4. 以下の情報を入力して「登録」ボタンをクリックする。
    - ユニークID: `testuser_01`
    - パスワード: `password` (強度が不足している)
    - 表示名: `テストユーザー01`
5. 「パスワードが短すぎます」「記号が含まれていません」等のバリデーションエラーが表示されることを確認する。
6. 以下の正しい情報を入力して「登録」ボタンをクリックする。
    - ユニークID: `testuser_01`
    - パスワード: `TestPassword123!`
    - 表示名: `テストユーザー01`
7. 登録が成功し、ログイン画面にリダイレクトされることを確認する。
8. 再度登録画面へ移動し、同じID `testuser_01` で登録を試みる。
9. 「このIDは既に使用されています」等の重複エラーが表示されることを確認する。

## 2. ログイン・ログアウトのテスト
1. ログイン画面で、誤ったパスワード `WrongPassword!` を入力してログインを試みる。
2. 「IDまたはパスワードが正しくありません」というエラーが表示されることを確認する。
3. 正しいログイン情報を入力して「ログイン」ボタンをクリックする。
    - ユニークID: `testuser_01`
    - パスワード: `TestPassword123!`
4. ログインに成功し、スレッド一覧画面が表示されることを確認する。
5. ブラウザの「ログアウト」ボタンをクリックする。
6. ログイン画面にリダイレクトされることを確認する。
7. ブラウザの「戻る」ボタンをクリックする。
8. 再度ログイン画面にリダイレクトされ、一覧画面が表示されないことを確認する。
