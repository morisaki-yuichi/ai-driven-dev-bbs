# 技術仕様書 (Technical Specification)

本プロジェクトのシステム構成、技術的な前提条件、および環境構築に関する定義を記述します。

## 1. 動作環境の前提条件

開発・動作させるローカルPC、またはコンテナ環境において、以下のツールがインストールされていることを前提とします。

1. **Git**: バージョン管理およびAIによるリポジトリ操作に必須です。
2. **Docker / Docker Compose**: データベースや各種ミドルウェアの構築に使用します。
3. **OS**: Ubuntu（WSLなど）または Windows。
   - ※Macでしか動作しないシステム（macOS固有のライブラリ依存等）は許可されません。
4. **パッケージマネージャ**: 技術スタックに応じた適切なツール。
   - 例: Node.js/TypeScriptなら `npm` / `yarn` / `pnpm` など。
   - 例: Pythonなら `pip` / `uv` など。

## 2. 技術スタックの選定方針

使用する言語、フレームワーク、ライブラリについては**完全に自由**です。AIエージェント自身に、作成された要件や設計を元に最適なスタックを選択させてください。

ただし、以下の制約を守る必要があります：
- **Webインターフェース**: アプリケーションはブラウザ上で動作する必要があります。
- **データ永続化**: データの保存方法（DB、ファイル、インメモリ等）はAIの技術選定に任せますが、永続化が行われるように実装してください。
- **外部依存の管理**: ローカルPCに直接ミドルウェア（PostgreSQL, Redis等）がインストールされていることは前提としません。必要な場合はDockerコンテナとして起動するように設定してください。
- **UI/UX**: AIが `agent-browser` を用いて動作確認を行うため、見えない要素にアクションを強いたり、極端に操作性が悪いデザインは避けてください（CSSの有無は自由です）。

## 3. インフラとデプロイ

- データベース等のインフラ構築手順（Dockerコマンド等）は、AIが自律的に発見・実行できるように、`README.md` またはそこから参照されるドキュメントにまとめてください。
