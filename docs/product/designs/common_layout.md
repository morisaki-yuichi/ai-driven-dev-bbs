# 共通レイアウト設計書

アプリケーション全体で統一して使用されるレイアウトコンポーネントおよび共通の仕様を定義します。

## 1. ヘッダー (Header)

全ての画面の上部に表示されます。ログイン状態により、右側の表示アクションが動的に切り替わります。

| 要素名 | 表示条件 | 内容 | アクション |
| :--- | :--- | :--- | :--- |
| アプリタイトル | 常に表示 | 「AI掲示板（仮）」 | クリックでトップページへ遷移 |
| ユーザー名 | ログイン中のみ | 「[表示名] さん」 | - |
| プロフィール | ログイン中のみ | リンク/ボタン | ユーザープロフィール編集画面へ遷移 |
| ログアウト | ログイン中のみ | ボタン | ログアウト処理を実行しログイン画面へ |
| ログイン | 未ログインのみ | リンク/ボタン | ログイン画面へ遷移 |
| 新規登録 | 未ログインのみ | リンク/ボタン | ユーザー登録画面へ遷移 |

## 2. フッター (Footer)

全ての画面の下部に表示されます。

| 要素名 | 内容 |
| :--- | :--- |
| コピーライト | © 2026 AI駆動開発教材プロジェクト |

## 3. 共通メッセージエリア (Notification Area)

画面上部または特定の入力エリア付近に表示される通知・エラーメッセージの仕様。

| 種類 | 表示内容の例 | スタイル（推奨） |
| :--- | :--- | :--- |
| 正常系通知 | 「保存しました」「削除しました」 | 緑系統の背景・テキスト |
| 異常系通知 | 「IDまたはパスワードが正しくありません」 | 赤系統の背景・テキスト |
| バリデーション | 「12文字以上で入力してください」 | 赤系統のテキスト（該当フィールド付近） |

## 4. 共通バリデーション・挙動ルール

- **未ログイン時**: 認証が必要なURL（/edit_profile等）にアクセスした場合は、一律でログイン画面へリダイレクトする。
- **削除後**: 存在しないページや削除済みリソースへのアクセスは、一律でエラー画面（404相当）を表示する。
