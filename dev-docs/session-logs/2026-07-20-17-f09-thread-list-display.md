  - G1（スコープ承認）: ユーザーが以下の範囲を承認した。

  - S2（証明+TDD実装）: 実装者が既存の不変条件 `hasNext_iff_more` が `p=0` において偽であることを発見し、反例を構成して確認したうえで修正・証明した（decision 0026 として起票）。
  - S3（レビュー）: 重大2件を検出・修正した。
    1. `comment_bumps_lastUpdated` も偽だった。`Db` にも `Wf` にも「既存レコードの `createdAt` は論理時計以下」という制約がなく、論理時計より未来の `createdAt` を持つ `db` で結論が破れる反例が存在する（`Wf` を満たすため `Wf` 側を足しても救えない）。`native_decide` で機械確認済み。局所仮定を追加して言明を真の形に直し、証明本体はF07の範囲として `sorry` を残した（decision 0027 として起票）。なお `sorted_by_commentCount` と `createdAsc_head_is_oldest` は検査の結果いずれも真だった。
    2. `?page=4294967295` で `next_page: page.page_number + 1` が算術オーバーフローし、ハンドラがpanicしていた（クエリ文字列のみでリクエストを落とせる状態）。`saturating_add` に修正し回帰テストを追加した。
    3. S3はdecision 0026の修正について「トートロジー化していない」と判定したが、`p=0` の1点で実装の追認に寄っているとの留保を付けた。
  - G3（裁定）: ユーザーが裁定し、以下を反映した。
    - decision 0026を承認したうえで、言明を「1 ≤ p」仮定の形へ変更し丸め規約を言明から追い出した（右辺は原典どおり `p * pageSize` に復帰。証明は無変更で通った）。
    - decision 0027を承認し、`Wf` への時計支配フィールドの集約はF07のサイクルで行う方針とした。
    - 範囲外ページに0件時とは別の文言を出すよう分離した。
    - 前後リンクが両方非表示のとき nav ごと出さないようにした。
    - `page` パラメータを `u32` 直接パースに修正した。
    - C-05テストを緩和した（`form`・`button`・`input` の禁止は維持、`a` は `href` が `/threads/<数字>` かつ `"edit"` を含まないことの検証に置換、編集エンドポイント非存在の検証は温存。実装側へのリンク追加はF10のスコープ）。
  - 裁定反映の検証として `params.rs` を一時的に旧実装へ戻して再ビルドする差分確認を行い、レビュー指摘より問題が広いことが判明した。実体は「`2^32 + k` が `k` ページ目として通る」ことで、`4294967297` はたまたま1ページ目に一致していただけだった。
  - 見送りが確定した事項: 日時整形をpaginateの後に移す最適化／テストヘルパの重複解消／間接検証を直接検証に変える件。
