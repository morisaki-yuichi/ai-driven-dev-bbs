# 0018 セットアップ・起動手順ドキュメントの配置とREADME参照方針

- 関連論点: D00
- 関連原典: H-06（README起点でのセットアップ・起動）、H-07（秘匿キー以外はAI自動実行）、H-13（clone直後から検証開始）
- 影響範囲: リポジトリ直下の新規ファイル配置、`README.md` の追記箇所、`foundation-plan.md` §2/§4.4/§5 との整合

## 背景（原典が何を言い、何を言っていないか）

- **[事実]** [decision 0001](0001-readme-editing-policy.md) が「手順の本文はREADME.mdに書かない。`docs/` の外の別ドキュメントに置き、README.mdからはリンクのみを追加する」「追加のみ・再構成なし」を既に確定している。
- **[事実]** `dev-docs/foundation-plan.md` §2 のディレクトリ構成案が、具体的なファイル名として**リポジトリ直下の `SETUP.md`**（`docs/` の外）を既に想定し、§4.4・§5 #14 もこの前提で書かれている。
- **[空白]** decision 0001 は「`docs/` の外」までしか規定しておらず、`docs/` の外のどこに置くか（リポジトリ直下か `dev-docs/` 配下か）は原典・既存decisionのどちらも明示していない。foundation-plan.md はAIの計画書であり、decisionとして番号を振った記録ではなかった。

## 選択肢

1. **リポジトリ直下 `SETUP.md`**（foundation-plan.md の既存想定を維持）。評価者・一般的なOSS利用者が最初に探す慣習的な場所であり発見性が高い。`dev-docs/` はAIの分析・決定記録の置き場という性格が強く、公開向けの運用手順とは性質が異なる。
2. **`dev-docs/` 配下**（例: `dev-docs/setup.md`）。CLAUDE.mdの「AI成果物はdev-docs/に置く」という原則に一貫させられる。ただし foundation-plan.md §2/§4.4/§5 の該当箇所を書き換える必要があり、変更コストが生じる。

## 提案（および理由）

選択肢1（リポジトリ直下 `SETUP.md`）を提案した。理由:

- decision 0001 の制約（`docs/` の外、本文はREADMEに書かない、リンクのみ追加）をどちらの選択肢も満たすため、この2択は制約からは一意に決まらない。
- 既存の `foundation-plan.md` が既にこの前提で複数箇所（§2 ディレクトリ構成、§4.4 README追記方針、§5 #14 実装順）を書いており、`dev-docs/` へ移すと計画書側の書き換えコストが生じる一方、得られる一貫性の便益は小さい。
- `SETUP.md` は評価者（人間・AI問わず）が実際に従う運用手順であり、`dev-docs/` に集約されているAIの分析・決定記録・作業ログとは性質が異なる。リポジトリ直下は多くのOSSで採用される慣習的な置き場所でもある。

## 決定（2026-07-19 ユーザー判断）

**選択肢1（リポジトリ直下 `SETUP.md`）を採用する。** `foundation-plan.md` の既存計画（§2 / §4.4 / §5 #14）を変更しない。

内容構成も既存計画を踏襲する: `SETUP.md` 1ファイルに「初回セットアップ」「アプリの起動」「評価をやり直す場合（空DBへのリセット、[0014](0014-thread-deletion-is-physical.md) と対をなす運用手順・D14）」の3セクションを設け、H-06 の2つの固定プロンプト（セットアップ／起動）にそれぞれ対応させる。`README.md` へは `SETUP.md` への参照リンクを1行のみ追加する（本文は書かない・既存記述は変更しない）。

`SETUP.md` の実際の作成とREADME.mdへのリンク追記は、`foundation-plan.md` §5 の実装順どおり**#1〜#13（compose.yaml・Dockerfile・migrations・`.sqlx/`・ドメイン層等）が揃ってから**行う（§5 #14）。現時点ではまだ検証可能なセットアップ手順が存在しない（`docker compose up` を通す実体が無い）ため、内容を先に書いても実測に基づかない記述になり、H-06充足の確認にならない。本decisionは配置と構成方針の確定であり、ファイル作成そのものは今回のセッションでは行わない。

## 影響

- `foundation-plan.md` は変更不要（この決定が既存計画を追認する形になったため）。
- 将来 `SETUP.md` を作成する際・`README.md` にリンクを追記する際は、本decisionと [0001](0001-readme-editing-policy.md) の両方の制約（追加のみ・本文は書かない・事後報告）に従う。
- `dev-docs/decisions/README.md` の一覧に本決定を追加する。
