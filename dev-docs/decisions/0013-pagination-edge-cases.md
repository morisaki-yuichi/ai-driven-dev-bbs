# 0013 ページネーションの境界（範囲外ページ・詳細ページ）

- ステータス: 決定済
- 起票日: 2026-07-19
- 関連論点: 新規（形式化により判明）
- 関連原典: C-12、AC09-3、AC09-5、C-10（404 の一律適用）
- 影響範囲: `Bbs.Query.paginate`、`Bbs.Query.threadDetail`

## 背景

`paginate` を書いた時点で、原典が答えていない境界が3つ出た。

- **[空白1]** **範囲外のページ番号**（`?page=999`、`?page=0`、`?page=-1`、`?page=abc`）を要求されたときの挙動。
  C-10 は「存在しないページ ⇒ 404」と言うが、これは**リソースの不在**を指しており、
  ページ番号の行き過ぎを含むかは読み取れない。
- **[空白2]** **0件のとき**の表示（1ページ目・前後ボタンとも非表示、で足りるか）。
- **[空白3]** **スレッド詳細のコメントにページネーションを掛けるか。** 原典は「関連する**全コメント**が
  表示される」（AC10-2）と言うので、掛けない方が素直。ただし AC11-3 の自動スクロールは、
  コメントがページ分割されているとヒット箇所が別ページに落ちて破綻する。

## 提案

1. **範囲外ページは空リストを返し、404 にしない。** モデルの `paginate` はこの挙動
   （`page = 0` は 1 として扱う）。理由: 一覧は常に存在するリソースであり、
   ページ番号は絞り込みパラメータにすぎない。404 にすると、削除でページ数が減った直後に
   有効だったURLが404になる。
2. **0件のときは1ページ目扱い**（`hasPrev = false`, `hasNext = false`）。空である旨のメッセージを出す。
3. **スレッド詳細のコメントはページネーションしない**（全件表示）。AC10-2 と AC11-3 の両方に素直。

## 決定（2026-07-19 ユーザー判断）

**提案どおり 1〜3 を採用する。**

1. **範囲外ページは空リストを返し、404 にしない。**

| 入力 | 挙動 |
| :--- | :--- |
| `?page=999` | 空の一覧（「前に戻る」のみ表示） |
| `?page=0` / `?page=-1` / `?page=abc` | 1ページ目扱い |

2. **0件のときは1ページ目扱い**（`hasPrev = false`, `hasNext = false`）。空である旨のメッセージを出す。
3. **スレッド詳細のコメントはページネーションしない**（作成日時昇順で全件表示）。

## 影響

### 404 にしないことの利点

スレッドを削除して総件数が 10 件以下になった直後、開いていた `?page=2` をリロードしても
404 にならない。一覧は常に存在するリソースであり、ページ番号は絞り込みパラメータにすぎない、
という整理。C-10 の「存在しないページ ⇒ 404」は**リソースの不在**を指すものと解釈する。

パース不能な値（`abc`）や負数も 1 ページ目に丸めるので、**ページ番号由来の 500 やクラッシュが起きない**。

### 詳細のコメント全件表示は AC11-3 の前提

コメントをページ分割すると、検索でヒットしたコメントが別ページに落ちて
**自動スクロール先が存在しなくなる**（AC11-3 が破綻する）。全件表示なら
`#comment-42` のアンカーが必ず同じページ内にあるので到達が保証される。

- decision 0012（タイトルを検索対象に含めない）と合わせ、スクロール先の場合分けは
  「本文（最上部・実質スクロール不要）」と「コメント（アンカー）」の2通りに収まる。
- コメント数が膨大になると重くなるが、評価規模では起きない。

### 形式モデル

`Query.paginate` は既にこの形（`page = 0` を 1 に丸め、範囲外は空リスト）。
`Query.threadDetail` もコメント全件を返す。変更不要。

なお**モデルは `Nat` を取るので負数・非数値は表現できない**。
「`?page=abc` を 1 に丸める」のは HTTP のパース層の責務であり、モデルの外側にある
（decision 0008 と同じく、レイヤの切り分けとして意識すること）。
