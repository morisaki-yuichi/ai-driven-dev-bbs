# 0005 表示名の空文字を許すか

- ステータス: 決定済
- 起票日: 2026-07-19
- 関連論点: D16
- 関連原典: C-03（表示名は15文字以内・重複可）、AC01-4、AC04-3、共通レイアウト `[表示名] さん`
- 影響範囲: `Bbs.Validation.displayNameFailure`、F01 登録、F04 プロフィール編集、ヘッダー表示

## 背景

- **[事実]** 原典が表示名について述べるのは「**15文字以内**」「重複可」「編集可能なのは表示名のみ」の3点。
- **[事実]** **下限（空を許すか）の記述がない。** 「15文字以内」を素直に読めば 0 文字も範囲内である。
- 空を許すと、ヘッダーの `[表示名] さん` が「 さん」になり、一覧の作成者欄も空欄になる。
  評価は AI が画面を自然言語で読んで判定するため、空欄は「作成者が表示されていない」（AC09-2 違反）と
  判定されうる。

## 提案

**空文字を許さない（1文字以上15文字以内）。** モデルでは `ValidationFailure.displayNameEmpty` を
`displayNameTooLong` とは別のエラーとして定義した。

理由: 原典は空を明示的に許してもいないので、安全側に倒す。AC09-2 の観測を壊すリスクを避ける。

## 決定（2026-07-19 ユーザー判断）

**空文字を許さない（トリム後 1〜15 文字）。** 登録・プロフィール編集の両方に同じ規則を適用する。

| 入力 | 判定 |
| :--- | :--- |
| `""` | `displayNameEmpty` |
| `"   "` / `"　"` | `displayNameEmpty`（decision 0004 のトリム後 0 文字） |
| `"a"` | OK |
| 15文字 | OK |
| 16文字 | `displayNameTooLong` |

## 影響

- decision 0004 と整合済み。`Validation.displayNameFailure` はトリム後の長さで判定し、
  0 文字と 16 文字以上を**別のエラー**として返す。
- **エラー文言は D11（decision 0006）の方針に従う**。原典は粒度のみ要求し文言は自由なので、
  「表示名を入力してください」「表示名は15文字以内で入力してください」のように
  違反理由が分かる文にする。空と長すぎを同じ文言にまとめないこと。
- ヘッダーの `[表示名] さん` と一覧の作成者欄が常に非空になる。AC09-2 の観測が安定する。
- 代替表示（「名前なし」等）を用意する必要はなくなった。
