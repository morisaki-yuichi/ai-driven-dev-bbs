# 0006 バリデーションの検査順序と複数エラーの提示

- ステータス: 決定済
- 起票日: 2026-07-19
- 関連論点: D11
- 関連原典: AC01-3、シナリオ01-1-5（`password` で「短すぎます」「記号が含まれていません」**等**）、`common_layout.md` 通知エリア
- 影響範囲: `Bbs.Op.register`、`Bbs.Validation.passwordWeaknesses`、フォームの表示

## 背景

- **[事実]** シナリオ01-1-5 は複数観点のエラーを**例示**する（「等」付き＝文言の完全一致は不要）。
- **[空白]** 複数の違反があるとき**全件を列挙するか最初の1件だけか**は未規定。
- **[空白]** 項目をまたぐ検査（ユニークID形式 → パスワード強度 → 表示名 → 重複）の**順序**も未規定。
  順序が違えば、複数項目が同時に不正なときに返るエラーが変わる。

`password` は「12文字未満」「数字なし」「記号なし」の**3違反**。最初の1件しか出さない実装だと、
AI が「記号が含まれていません」を確認できず、シナリオが不合格になるおそれがある。

## 提案

1. **パスワード強度違反は全件を列挙する。** モデルでは `passwordWeaknesses : String → List PasswordWeakness`
   が違反の内訳（`tooShort` / `noAlpha` / `noDigit` / `noSymbol` / `disallowedChar`）をリストで返す。
2. **項目をまたぐ検査は「形式 → 強度 → 表示名 → ID重複」の順**で、最初に失敗した項目のエラーを返す。
   重複チェックを最後に置くのは、DB アクセスを伴うため。
3. 表示形式は「該当フィールド付近に赤系テキスト」（`common_layout.md`）に従い、
   パスワード欄には違反理由を**箇条書きで全件**出す。

## 決定（2026-07-19 ユーザー判断）

**提案どおり 1〜3 を採用する。**

1. **パスワード強度違反は全件列挙**（`passwordWeaknesses` が内訳をリストで返す）。
2. **項目間は「形式 → 強度 → 表示名 → ID重複」の順で、最初に失敗した項目のエラーのみ**を返す。
3. **ID重複チェック（DB アクセス）は形式検査の後ろ**に置く。形式が全部正しいときだけ DB を引く。

### 具体例

入力 `ID="" / パスワード="password" / 表示名=16文字` のとき:

```
ユニークID  [        ]
  ※ ユニークIDを入力してください
（他の項目のエラーは、ID を直すまで見えない）
```

シナリオ01-1-5 は ID と表示名を正しく入れてパスワードだけ `password` にするため、
**3違反がそのまま全件表示される**（この決定でシナリオは通る）:

```
パスワード  [        ]
  ※ 12文字以上で入力してください
  ※ 数字を含めてください
  ※ 記号を含めてください
```

## 影響

- 形式モデルは既にこの形（`Bbs/Op.lean` の `register`）。変更不要。
- `Error.validation` は単一の `ValidationFailure` を持つ形のままでよい
  （全項目同時表示を選んでいたら `List ValidationFailure` に直す必要があった）。
- **エラー文言は違反理由が分かる粒度で個別に用意する**（原典は文言の完全一致を要求していない）。
  以下は最低限の対応表。文言自体は変えてよいが、**理由を潰して1文にまとめないこと**。

| `ValidationFailure` / `PasswordWeakness` | 文言例 |
| :--- | :--- |
| `uniqueIdInvalid` | ユニークIDを入力してください |
| `tooShort` | 12文字以上で入力してください |
| `noAlpha` | 英字を含めてください |
| `noDigit` | 数字を含めてください |
| `noSymbol` | 記号を含めてください |
| `disallowedChar` | 使用できない文字が含まれています（使えるのは英数字と `!@#$%^&*()_+-=[]{}\|;':",./<>?` です） |
| `displayNameEmpty` | 表示名を入力してください |
| `displayNameTooLong` | 表示名は15文字以内で入力してください |
| `titleEmpty` / `bodyEmpty` | タイトル／本文を入力してください |

- 表示位置は `common_layout.md` に従い「該当フィールド付近に赤系テキスト」。
  パスワード欄には違反理由を**箇条書きで全件**出す。
