# 0001 README.md へのセットアップ手順追記の可否

- ステータス: 決定済
- 起票日: 2026-07-19（決定日 2026-07-19、session#03 の判断を転記）
- 関連論点: D00
- 関連原典: H-06（README 起点で setup・起動）、`docs/guide/development_rules.md` §1、`docs/product/technical_spec.md` §3
- 影響範囲: リポジトリ運用全般

## 背景

- **[事実]** README §3 と technical_spec §3 は、セットアップ・起動手順を「`README.md` またはそこから参照されるドキュメント」にまとめることを要求する（H-06）。
- **[事実]** `development_rules.md` が禁じるのは**人間**の操作のみで、AI による README/docs の編集を禁じる記述は原典に無い。原典レベルでの衝突は存在しない。
- **[運用]** 一方、セッション指示に「`docs/` 配下と README.md は読み取り専用」があり、その射程が未確定だった。

## 決定（2026-07-19 ユーザー判断）

README.md も原則編集しないが完全な不可侵ではない。H-06 充足のため、**手順への参照追加に限った最小限の追記**を、人間の確認のうえ認める。

派生する運用制約（要件分析 §2.3 の **P-08**）:

- 手順の**本文**は README.md に書かない。別ドキュメントに置き、README.md からは**リンクのみ**を追加する。
- 手順ドキュメントの置き場所は `docs/` の**外**（`docs/` は引き続き読み取り専用）。
- **追加のみ・再構成なし**（既存記述の削除・書き換え・並べ替えを行わない）。
- この範囲に収まる限り**事前承認は不要**（事後報告で足りる）。範囲を超える変更は事前承認を要する。

## 影響

- H-06 は達成可能になった。フェーズ0 の最優先ブロッカーは解消済み。
- 手順ドキュメントの具体的な置き場所・ファイル名は未定（技術スタック選定後に決める）。
