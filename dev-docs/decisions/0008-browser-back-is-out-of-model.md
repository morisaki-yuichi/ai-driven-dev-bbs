# 0008 AC03-2（ブラウザバック）はデータモデル外の要件

- ステータス: 決定済
- 起票日: 2026-07-19
- 関連論点: D01, D04
- 関連原典: AC03-2、C-11
- 影響範囲: HTTP レスポンスヘッダ、レンダリング方式の選定

## 背景

形式化の過程で、**AC03-2 は状態機械としては表現できない**ことがはっきりした。

- **[事実]** AC03-2「ログアウト後、ブラウザの『戻る』を押してもログイン画面へリダイレクトされる」。
- モデル上、ログアウト後はセッションが消えるので `viewThreadList` は必ず `notAuthenticated` になる
  （`Invariant.viewThreadList_requires_auth` として言明済み）。**サーバ状態としては既に正しい。**
- それでもブラウザは**キャッシュ済みの HTML** を戻るボタンで再表示しうる。これはサーバ状態と無関係で、
  `Cache-Control: no-store` 等の**レスポンスヘッダの問題**である。

## 提案

- AC03-2 を「データモデルの不変条件」ではなく「**HTTP レイヤの要件**」として分類し、
  認証必須ページに `Cache-Control: no-store, must-revalidate` を付ける方針とする。
- 形式モデルはこの要件を**カバーしない**ことを明記する（`Bbs.Op.logout` のコメントに記載済み）。
  モデルの証明が全部通っても AC03-2 は保証されない。

## 決定（2026-07-19 ユーザー判断）

1. AC03-2 を**データモデルの不変条件ではなく HTTP レイヤの要件**として分類する。
   形式モデルはこの要件を**カバーしない**（`Bbs/Op.lean` の `logout` に明記済み）。
   モデルの証明が全部通っても AC03-2 は保証されない。
2. 認証必須ページに `Cache-Control: no-store, must-revalidate` を付ける。
3. **AC03-2 をレンダリング方式の選定制約として扱う。フェーズ1 の候補はサーバレンダリング
   （SSR / MPA）に絞る。SPA は候補から外す。**

## 影響（フェーズ1 の技術選定を拘束する）

これは形式化で出た決定のうち、**技術選定を直接縛る唯一のもの**である。

- **候補は SSR / MPA に限定される。** SPA を選ぶと、戻るボタンがサーバへのリクエストを伴わないため
  ヘッダでは防げず、クライアント側で「ログアウト時に認証状態を確実に破棄」「各ルートでガードを再評価」
  「メモリ上のキャッシュ済み応答も破棄」をすべて自前で実装する必要がある。
  評価は agent-browser で**実際に戻るボタンを押す**ため、取りこぼすと即座に不合格になる。
- SSR/MPA なら**認証必須ページにヘッダを1行**足すだけで済む。
- **副次的な恩恵**: AC07-3「投稿後、即座にコメント一覧に反映」も POST-Redirect-GET で素直に満たせる。
  `ui_design.md` が P04 の送信ボタンについて「投稿後、**P04を再読み込み**」と明記しているのとも整合する（D06）。
- D01（技術スタック）はこの決定により**選択肢が狭まった**。`dev-docs/requirements-analysis.md` §3 D01 に
  相互参照を追記した。
