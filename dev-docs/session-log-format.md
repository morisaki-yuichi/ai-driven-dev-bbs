# セッションログ形式定義

`dev-docs/session-logs/` に残す作業記録の形式を定義する。以降のセッションは本ファイルを参照して形式を揃えること。

## ファイル名規約

```
dev-docs/session-logs/<日付>-<連番>-<スラッグ>.md
```

- `<日付>`: `YYYY-MM-DD` 形式（作業日）。
- `<連番>`: そのプロジェクトを通した通し番号を 2桁ゼロ埋めで表記（`01`, `02`, ...）。同日に複数セッションがある場合も番号は継続する。
- `<スラッグ>`: 内容を表す英小文字ケバブケース（例: `phase0-analysis`, `phase1-stack-selection`）。

例: `dev-docs/session-logs/2026-07-19-01-phase0-analysis.md`

## 見出し構成

```markdown
# セッションログ: <日付> #<連番> <タイトル>
- フェーズ: <フェーズ番号>
- 今回やったこと:
- 決めたこと（関連 decision 番号があれば併記）:
- 次にやること:
- 未解決事項:
```

## 記述ルール

- 5項目はすべて必須。該当する内容がない場合は「なし」と明記し、項目自体を省略しない。
- 各項目の内容が複数ある場合は、項目行の下にネストした箇条書きで列挙する。
- 「決めたこと」に decision 記録（`dev-docs/decisions/` 等）が対応する場合は、決定内容の末尾に番号を併記する（例: `（decision 0001）`）。
- 「未解決事項」には、次セッション以降に持ち越す論点や、ユーザー判断待ちの事項を書く。要件分析の論点ID（`D00`〜）がある場合は併記する。
- 日付は相対表現（「今日」「昨日」）を使わず、常に絶対日付で書く。
