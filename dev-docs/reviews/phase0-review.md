## 重大

### R-01 「README.md／docs/ はAIも編集禁止」を[事実]として扱っているが、原典からは導けない（D00・P-03・harness-invariants H-06⚠️に伝播）

- 対象箇所: `requirements-analysis.md` D00（「**[事実]** 本セッションの指示により README.md は読み取り専用で編集禁止」）、P-03（「人間も**AIも**編集しない」）、`harness-invariants.md` H-06 の「⚠️ 未解決の衝突」注記。
- 原典の実際の記述: `docs/guide/development_rules.md` が禁じているのは**人間**の操作のみである（§1「人間が行うことを禁止します」、ドキュメント編集も「人間が行うことは避けてください」）。AIによる README.md・docs/ の編集を禁じる記述は原典のどこにも存在しない。README.md はそもそも `docs/` 配下ですらない。
- さらに、原典はむしろ README の更新を**成果物として要求**している: `technical_spec.md` §3「インフラ構築手順は…README.md またはそこから参照されるドキュメントにまとめてください」。現状の README から新設ドキュメントへの参照経路は存在しないため、H-06 を満たすには README 編集が必然であり、これは本教材の想定された作業（AIがREADMEにセットアップ手順を書く）そのものと読める。**原典レベルでは D00 の「衝突」は存在しない。**
- 問題の構造: 編集禁止の根拠は「本セッションの指示」という原典外の情報だが、これが [事実] ラベルで書かれ、「AIが単独で解決できない唯一の論点・最優先・ユーザー判断が必要」という位置づけで `harness-invariants.md`（後続全フェーズの照合基準）にまで警告として伝播している。もしセッション指示の実際の射程が「フェーズ0の分析作業中は原典を書き換えない」であったなら、恒久制約への拡大解釈が下流全体の進行を誤ってブロックする。逆に恒久制約が本当なら D00 は正しい。**どちらかをフェーズ1前に確定させる必要がある。**
- 併せて D00 選択肢(c)の「technical_spec も読み取り専用のため手順を書けない（リスク大）」も同じ前提の上に立っており、前提が崩れれば評価が変わる。
- 推奨: セッション指示の原文を確認し、(1) 読み取り専用の対象範囲（README を含むか）と (2) 期間（フェーズ0限定か恒久か）を明記した上で、D00 を「原典上は衝突なし・運用指示との衝突の有無のみ要確認」に書き直すか、恒久制約の典拠を追記する。

## 中

### R-02 AC10-3 の転記で原文の許容句を脱落させ、C-07 を原文より強い固定制約にしている

- `docs/product/issues/10_thread_detail_display.md` の原文AC: 「本来の作成者や日時は維持されていること**（または仕様に合わせて適切に表示されていること）**」。対象ファイルの AC10-3 はこの括弧書きを落とし「維持されている」のみを太字で断定、C-07 でも「作成者・作成日時は維持し」を固定制約として記載している。
- 原文は代替表示を明示的に許容しており、「維持」は必須ではない。安全側への倒し方としては合理的だが、凡例上これらは [事実] 扱いであり、原文より強い要求を事実として下流に渡している。[解釈]（採用方針）として明示すべき。

### R-03 C-14 が、D08 で「要決定」とした解釈を §2.2 の確定制約として先取りしている（内部不整合）

- C-14: 「**論理削除で残っている元本文も検索にヒットしてはならない**」と断定。一方 D08 は同じ内容を「原文が二重に解釈できる／②が必要条件と読むのが妥当。**要決定**」と論点登録している。
- 原文 AC11-4 は「検索対象にならない（**あるいはヒットしても遷移時に矛盾がない**）ように制御」であり、後者の実装（例: ヒット時に削除済み位置へスクロールして矛盾なく見せる）も許容している。§2.2 は「実装が必ず守るべき事項」＝事実側のセクションなので、未決の解釈をここに置くのは切り分けの誤り。C-14 を「AC11-4 の充足（方式は D08 で決定）」程度に弱めるべき。

### R-04 カバレッジ表 #24 の出典説明が不正確（F13 の規定は issues/09 にほぼ全文存在する）

- 表 #24（misc メモ）の備考: 「F13ページネーションの詳細規定（10件・前後ボタン必須・端で非表示・任意項目）は**このファイルにのみ全文が存在**」。
- 実際には `issues/09_thread_list_display.md` に 1ページ10件・「前に戻る」「次に進む」必須・1ページ目/最終ページでの非表示・任意項目（件数切替、全件数、ページ番号ジャンプ）が**すべて**記載されている。メモにのみ存在するのは「前後ボタンのラベル・表示は任意でよい」という一点のみ。F13 本文の内容自体は正しいが、出典の主張が誤っており、原典監査時に誤導しうる。
